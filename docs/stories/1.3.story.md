## Context

Users need to browse AMC domains, topics, and subtopics to choose what to study. The domain navigator in the left panel is the primary navigation system — it shows progress, mastery levels, and lets users filter their study sessions.

## Acceptance Criteria

- [ ] **AC1** — `<DomainNavigator>` in left panel shows all 6 AMC domains from DB:
  - Domain name + icon + color dot
  - Question count per domain
  - User's mastery level badge (Novice → Fellow)
  - Accuracy percentage (from `user_domain_mastery`)
- [ ] **AC2** — Collapsible tree: Domain → Topics (click to expand)
- [ ] **AC3** — Active domain/topic highlighted with cinnamon accent
- [ ] **AC4** — "Study this domain" button on hover triggers filtered practice session
- [ ] **AC5** — `<RightPanel>` shows performance overview:
  - FSRS cards due today (count with urgency color)
  - Daily goal progress bar (animated)
  - Current streak with flame icon
  - "Weakest area" — lowest accuracy domain
  - Quick action: "Start session" CTA always visible
- [ ] **AC6** — Domain navigator collapses to icon-only mode on smaller screens (>768px but compressed)
- [ ] **AC7** — Empty state: if no questions studied yet, show "Begin your journey" onboarding prompt
- [ ] **AC8** — Topic completion percentage (questions answered / total in topic)

## Technical Notes

- Data fetched server-side (RSC) for initial load — no loading spinner on first paint
- Domain data from `subjects` + `user_domain_mastery` joined
- Topic data lazy-loaded on expand (client-side fetch)
- `<RightPanel>` stats refresh every 60s via React Query polling
- Mastery level colors match domain colors from design system
