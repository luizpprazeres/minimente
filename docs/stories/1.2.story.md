## Context

The FSRS-5 spaced repetition engine is the learning backbone of miniMENTE. It schedules which questions to show, when to show them, and updates difficulty/stability after each review. This story implements the TypeScript FSRS engine and the full study session loop.

## Acceptance Criteria

- [ ] **AC1** — `src/lib/srs/fsrs.ts` implements FSRS-5 algorithm:
  - `scheduleCard(card, grade, now)` → returns updated card with new S, D, R, due_date
  - `calcRetrievability(elapsedDays, stability)` → R(t,S) = (1 + F·t/S)^C
  - `calcInterval(stability, targetRetention)` → interval in days
  - All 19 FSRS-5 default parameters (w0–w18) implemented
  - Handles all 4 states: new → learning → review → relearning
- [ ] **AC2** — `useStudySession` hook manages the session loop:
  - Fetches due cards from `get_due_cards()` DB function
  - Advances to next question after answer
  - Saves review to `review_log` table after each answer
  - Updates `fsrs_cards` after each review
  - Handles session end (no more due cards)
- [ ] **AC3** — Practice mode page (`/workspace/practice`) is functional end-to-end:
  - Shows question → user answers → FSRS updates → next question
  - Session persists across browser refreshes (stored in DB)
- [ ] **AC4** — XP awarded per correct answer (+10 base, +25 for hard questions)
- [ ] **AC5** — Streak updated: if first session of the day, increment streak
- [ ] **AC6** — Session summary screen after completing a session:
  - Questions answered, accuracy %, XP earned, streak status
  - Animated results reveal (stagger each stat)
- [ ] **AC7** — "Due today" count shown in right panel (from `fsrs_cards` where `due_date <= now()`)
- [ ] **AC8** — When a user answers a question for the first time (state = 'new'), create FSRS card and insert first review

## Technical Notes

- FSRS-5 default weights (w0–w18) from the research doc
- All FSRS calculations happen client-side (in the browser) — sync to DB after each grade
- Use Supabase `upsert` for `fsrs_cards` (insert on first review, update on subsequent)
- `review_log` is append-only — never update, always insert
- XP update: use Supabase RPC function or direct UPDATE with optimistic UI
- If no `fsrs_cards` exist yet for user: system picks from published questions (newest first)
