# Story 0.1 — Design System Foundation & App Shell

**Epic:** Phase 0 — Foundation
**Status:** Done
**Agent:** @dev (Dex)

## Context

miniMENTE needs a solid design system and app shell before any feature work begins. This story establishes the visual identity (Cinnamon theme), base layout components, and the single-workspace structure that all subsequent stories will build upon.

## Acceptance Criteria

- [x] **AC1** — Tailwind CSS v4 tokens fully configured (`globals.css`) with all Cinnamon palette values, warm neutrals, semantic colors, and AMC domain colors
- [x] **AC2** — `cn()` utility function available (`src/lib/utils.ts`) using `clsx` + `tailwind-merge`
- [x] **AC3** — Base layout components created:
  - `<AppShell>` — 3-panel workspace layout (sidebar 280px / main / panel 320px)
  - `<Header>` — logo + streak + XP + avatar slots
  - `<LeftPanel>` — domain navigator skeleton
  - `<RightPanel>` — performance overview skeleton
- [x] **AC4** — Framer Motion installed and `MotionConfig` provider wrapping app (reduce-motion aware)
- [x] **AC5** — Google Fonts loaded: Lora (display) + Inter (body) with `next/font` — zero layout shift
- [x] **AC6** — Responsive breakpoints working: mobile (<768px) collapses to single column with bottom nav
- [x] **AC7** — Dark mode CSS variables defined and togglable via `data-theme` attribute
- [x] **AC8** — `npm run build` passes without errors, app routes at `/dashboard`

## Technical Notes

- Use Tailwind v4 `@theme` block (already in `globals.css`) — no separate `tailwind.config.ts` needed
- All layout components in `src/components/layout/`
- Use CSS Grid for 3-panel layout (`grid-cols-[280px_1fr_320px]`)
- Framer Motion `AnimatePresence` should wrap route transitions at root level
- Bottom nav for mobile: Study / Domains / Progress / Vocab / Profile (5 tabs)
- i18n config added: `src/i18n/request.ts` + `messages/en.json` + `messages/pt.json`
