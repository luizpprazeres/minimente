## Context

The question card is the core interaction of miniMENTE. It must feel alive — with animations for entry, selection, correct/incorrect feedback, and explanation reveal. This story builds the complete question answering experience.

## Acceptance Criteria

- [ ] **AC1** — `<QuestionCard>` component renders: stem text, optional media slot, 5 answer options (A–E)
- [ ] **AC2** — Answer options animate through states: `default → hover → selected → revealed-correct / revealed-incorrect`
  - Correct: green border pulse + checkmark bounce animation (400ms)
  - Wrong: red background + subtle shake (300ms) + shows correct answer highlighted
- [ ] **AC3** — Question card entry animation: slide-in from right + fade, spring physics (stiffness 300, damping 30)
- [ ] **AC4** — Keyboard navigation: press A/B/C/D/E to select, Enter to confirm, Space/→ for next
- [ ] **AC5** — `<ExplanationPanel>` accordion opens below card after answer submitted:
  - Key Concept highlighted in cinnamon
  - Full explanation text
  - Reasoning steps (numbered, animated stagger)
  - Source references as chips
- [ ] **AC6** — Progress bar at top shows `n/total` questions in session (animated fill)
- [ ] **AC7** — "Next question" button with arrow — auto-advances after 3s if explanation shown
- [ ] **AC8** — Session timer visible (counts up, or counts down in exam mode)
- [ ] **AC9** — Question renders correctly with media: ECG image, X-ray placeholder, clinical photo
- [ ] **AC10** — Bilingual: question renders in user's `language_pref`, can toggle EN↔PT inline

## Technical Notes

- All animations via Framer Motion (`motion.div`, `AnimatePresence`)
- Spring config: `{ type: "spring", stiffness: 300, damping: 30 }`
- `<QuestionCard>` receives a `Question` type from `@/types/database`
- Answer state machine: `idle → selected → confirmed → revealed`
- Use `useReducer` for answer state (not useState — too many transitions)
- Explanation panel uses Framer Motion `height: "auto"` animation
- A11y: `role="radiogroup"`, each option is `role="radio"`, keyboard fully navigable
