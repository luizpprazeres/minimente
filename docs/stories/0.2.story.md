# Story 0.2 — Supabase Auth + User Profile Flow

**Status:** Done

## Context

Users need to sign up and sign in before accessing any study content. This story implements the full auth flow using Supabase Auth with email/password and Google OAuth, plus the auto-created user profile (trigger already in DB).

## Acceptance Criteria

- [ ] **AC1** — Sign up page (`/auth/signup`) with email + password fields, bilingual labels (EN/PT switch)
- [ ] **AC2** — Sign in page (`/auth/login`) with email/password + "Continue with Google" button
- [ ] **AC3** — Password reset flow (`/auth/reset-password` + `/auth/update-password`)
- [ ] **AC4** — Auth middleware (`src/middleware.ts`) protects all `/workspace/*` routes, redirects unauthenticated users to `/auth/login`
- [ ] **AC5** — After successful signup, DB trigger auto-creates: `user_profiles`, `user_settings`, `user_xp`, `user_streaks`, `user_domain_mastery` (6 rows)
- [ ] **AC6** — Onboarding screen after first login: select language (EN/PT) + set daily goal (10/20/40 questions)
- [ ] **AC7** — User session persists across page refreshes (Supabase SSR cookies)
- [ ] **AC8** — Sign out button clears session and redirects to `/auth/login`
- [ ] **AC9** — Auth errors show inline (not alert boxes): "Email already in use", "Invalid credentials", etc.

## Technical Notes

- Use `@supabase/ssr` (already in package.json) for cookie-based session management
- Middleware reads cookies from `request.cookies` — no localStorage
- Google OAuth needs configuration in Supabase Dashboard → Auth → Providers
- After OAuth callback, check if `onboarding_done = false` → redirect to `/onboarding`
- All auth pages use the Cinnamon design (warm background, logo prominent)
- Form validation with `zod` (already installed)
